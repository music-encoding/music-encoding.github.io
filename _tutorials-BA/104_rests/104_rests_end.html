Congratulations!

In this tutorial, you learned how to encode rests with MEI. The elements you've learned are:

  • <rest>
  • <mRest>
  • <multiRest>
  • <space>
These elements are almost always empty and have no child elements. With the space element, you already learned about a fairly advanced mechanism of aligning multiple voices sharing a staff in MEI. If you're interested in that, you may also want to investigate into the @next and @prev attributes from MEI's att.linking attribute class (available on <note> and other events). That attribute class allows to lay "breadcrumbs" through an MEI file that make it possible to follow voices across multiple layers (and staves, if necessary). This would facilitate analytical use of your MEI file, and is not required in most situations, though.
